# /self-improve — 메타인지적 자기 개선

세션 평가 데이터를 분석하고, 전략을 제안하고, **사용자 승인 후에만** CLAUDE.md에 반영한다.

## 실행 순서

### 1단계: 분석 실행

```bash
python3 ~/projects/self-improving-skill/src/memory.py analyze
```

### 2단계: 전략 검증

```bash
python3 ~/projects/self-improving-skill/src/verifier.py
```

검증 결과를 읽고 각 전략의 상태를 파악한다:
- `verified` — 적용 전보다 지표가 나아짐. 적용 권장.
- `unverified` — 적용 후 지표가 나빠짐. 제거 권장.
- `neutral` — 효과 불분명. 사용자 판단 필요.
- `pending` — 데이터 부족. 아직 판단 불가.

### 3단계: 제안 파일 확인

현재 프로젝트의 `.self-improve-strategies.md`를 읽고 사용자에게 보여준다.

### 4단계: 리포트 + 승인 요청

사용자에게 다음 형식으로 보고한다:

```
## 세션 분석 리포트

**분석 범위**: N개 세션 / 프로젝트: {name}

### 잘하고 있는 것
- (구체적 근거와 함께)

### 제안된 전략
1. [검증됨] "전략 내용" — 근거: ...
2. [대기] "전략 내용" — 아직 데이터 부족

### 감지된 안티패턴
- 패턴명 (x횟수) — 제안

---
이 전략들을 CLAUDE.md에 적용할까요?
- 전체 적용
- 선택 적용 (번호 지정)
- 적용 안 함
```

**원칙:**
- 개선점은 최대 2개만 제시
- "잘하고 있는 것"을 먼저 인정
- 추상적 조언 금지 → 구체적 행동 제안
- **사용자가 승인하지 않으면 CLAUDE.md를 절대 수정하지 않는다**

### 5단계: 승인 시 적용

사용자가 승인한 전략만 프로젝트의 CLAUDE.md 맨 끝에 마커 블록으로 추가한다:

```markdown
<!-- self-improve:start -->
## 자기 개선 전략 (승인됨)

- 전략 1
- 전략 2

<!-- self-improve:end -->
```

기존 마커 블록이 있으면 교체, 없으면 맨 끝에 추가.
마커 밖의 기존 CLAUDE.md 내용은 절대 수정하지 않는다.
