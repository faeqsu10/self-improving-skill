# /self-improve — 메타인지적 자기 개선

세션 평가 데이터(scores.json)를 분석하고, 작업 패턴에서 개선점을 찾아 전략(memory.json)으로 저장한다.

## 실행 순서

### 1단계: 데이터 수집

```bash
python3 ~/projects/self-improving-skill/src/memory.py analyze
```

출력된 JSON을 읽고 핵심 발견을 파악한다.

### 2단계: 메타 분석

분석 결과를 바탕으로 다음을 판단한다:

**프로젝트별 상태 진단:**
- 테스트 커버리지가 있는 프로젝트 vs 없는 프로젝트
- lint 에러가 쌓이고 있는 프로젝트
- 활발한 프로젝트 vs 방치된 프로젝트

**안티패턴 해석:**
- 감지된 안티패턴이 실제로 문제인지 맥락적으로 판단
- "테스트 없이 작업"이 프로토타입 단계라 괜찮은 건지, 진짜 위험한 건지

**추세 해석:**
- 개선되고 있는 지표 → 무엇이 효과적이었는지 기록
- 퇴보하는 지표 → 원인 추론

### 3단계: 전략 제안

분석 결과를 사용자에게 보고한다. 형식:

```
## 세션 분석 리포트

**분석 범위**: N개 세션 / M개 프로젝트

### 잘하고 있는 것
- (구체적 근거와 함께)

### 개선 포인트 (최대 2개)
1. **[패턴명]**: 설명 + 구체적 행동 제안
2. **[패턴명]**: 설명 + 구체적 행동 제안

### 다음 세션에서 시도할 것
- (1개만, 구체적으로)
```

**원칙:**
- 개선점은 최대 2개만 (HyperAgents도 한 세대에 과도한 수정을 하면 성능이 떨어졌다)
- "잘하고 있는 것"을 먼저 인정한 후 개선점 제시
- 추상적 조언("테스트를 잘 작성하세요") 금지 → 구체적 행동("다음 세션에서 코드 수정 전 pytest -x를 먼저 돌려보세요")

### 4단계: Memory 확인

```bash
python3 ~/projects/self-improving-skill/src/memory.py summary
```

memory.json이 제대로 갱신되었는지 확인하고, 필요하면 사용자에게 전략 조정을 제안한다.
